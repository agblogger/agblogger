# E2E Testing Review - March 10, 2026

Comprehensive end-to-end testing of AgBlogger using Playwright MCP, covering all screens, controls, user workflows, edge cases, visual aesthetics, and accessibility.

## Bugs Found

### 1. Editor preview shows placeholder on initial load

- **Severity**: Low
- **Location**: Editor page (`/editor/{file_path}`)
- **Description**: When navigating to an existing post's editor, the preview pane shows "Start typing to see a live preview" even though content is present in the textarea. Clicking the textarea triggers the preview to render correctly.
- **Steps to reproduce**: Navigate directly to `/editor/posts/<slug>/index.md` for an existing post. Observe the preview pane.
- **Expected**: Preview should render immediately when editor loads with existing content.
- **Suggestion**: Trigger preview render on editor mount when content is already loaded, not only on user interaction.

### 2. Label combobox shows "Create" for existing labels

- **Severity**: Medium
- **Location**: Editor label combobox (`/editor/*`)
- **Description**: When typing a label name that already exists (e.g., "swe"), the combobox dropdown shows "Create #swe" instead of showing the existing label as a selectable option. Selecting it causes a 409 Conflict from the API (attempting to create a duplicate label), though the label is still assigned to the post.
- **Steps to reproduce**: Open the editor for any post. Type an existing label ID in the label combobox. Observe the dropdown suggestion.
- **Expected**: The combobox should differentiate between selecting an existing label and creating a new one. Existing labels should appear as selectable suggestions.
- **Suggestion**: Pre-populate combobox suggestions from existing labels. Show "Create #xyz" only when the typed ID does not match any existing label.

## Visual/UX Assessment

### Overall Design Quality: Very Good

The application has a distinctive, warm aesthetic that avoids generic "AI-designed" looks.

### Design Strengths

- **Color palette**: Warm cream/beige backgrounds with terracotta/rust accent for primary actions. Cohesive editorial feel that is distinctive and pleasant.
- **Typography**: Serif headings with clean sans-serif body text create a nice editorial/magazine hierarchy. The blog title in the header uses a refined serif that sets the tone.
- **Spacing & layout**: Generous whitespace throughout. Content is well-contained with comfortable reading widths. Post view has excellent typographic rhythm.
- **Component design**: Cards, buttons, and form elements have consistent rounding and subtle shadows. Label pills, filter chips, and tab controls all feel cohesive.
- **Label graph visualization**: Dagre auto-layout, mini-map, zoom controls, and node cards showing label info at a glance.
- **Rich excerpt rendering**: Timeline excerpts render math, formatted text, and labels inline.
- **Split-pane editor**: Live Pandoc preview is functional and well-implemented.
- **Dark mode**: Well-executed with good contrast and appropriate color shifts for all elements including code blocks and math.
- **Share dropdown**: Comprehensive with proper platform icons (Bluesky, Facebook, Hacker News, LinkedIn, Mastodon, Reddit, X, Email, Copy link, device share).
- **TOC navigation**: Clean popover approach with scroll tracking.
- **Code blocks**: Language labels + copy-to-clipboard buttons.
- **Filter panel**: OR/AND toggle, label search, date range, and author filter is powerful and well-organized.

### Suggestions for Improvement

1. **`<mark>` highlight in dark mode**: The bright yellow `<mark>` background is harsh against the dark theme. Consider a softer amber/gold with reduced opacity in dark mode.

2. **Editor toolbar discoverability**: The formatting toolbar (Bold, Italic, Heading, Link, Code, Code Block) uses icon-only buttons. While keyboard shortcuts are in tooltips, adding text labels or more visible tooltips would help new users.

3. **Empty timeline state**: When no posts exist, the timeline shows only upload buttons with no welcome message. A friendly empty state ("No posts yet. Write your first post!") would improve the first-run experience.

4. **Admin Settings display name field**: Architecture docs mention a display name field in admin settings, but it was not visible in the Settings tab. Either add the field or update the docs.

5. **Filter panel label counts**: Label filter buttons show counts in a concatenated format (e.g., "#swe1"). A space or badge would be clearer (e.g., "#swe (1)").

### Detailed Test Coverage

#### Authentication
- Login with valid credentials (admin/admin) redirects to timeline
- Login with invalid credentials shows error message
- Login with empty fields shows validation
- Logout redirects to login page
- Session recovery after page refresh

#### Post Creation & Complex Markdown
- Created post with all Pandoc markdown features:
  - Bold, italic, strikethrough, inline code, highlighted text (`==mark==`)
  - Tables with formatted cells
  - Inline and display math (KaTeX): `$E=mc^2$`, `$$\int_{-\infty}^{\infty}$$`
  - Python and JavaScript code blocks with syntax highlighting
  - Nested blockquotes
  - Ordered, unordered, and checkbox lists with deep nesting
  - Internal heading references (`[Introduction](#intro)`)
  - Unicode: Japanese, Chinese, Korean, Greek, Arabic, Russian, emoji
  - Definition lists, footnotes, horizontal rules, superscript/subscript

#### Image Upload & File Management
- Uploaded two test images via file chooser
- File strip shows thumbnails and file count
- Insert adds markdown image syntax at cursor
- Rename updates file reference

#### Label Management
- Created labels via API (cs, swe, math, physics)
- Set parent relationships (swe -> cs, swe -> math)
- Verified list view shows labels with post counts and parent info
- Graph view visualizes DAG with Dagre layout and edge rendering
- Label settings: add/remove display names, toggle parent checkboxes
- Cycle prevention: descendants excluded from parent options with explanatory text
- Cycle detection via API returns 409 with clear error message
- Label deletion with confirmation dialog, redirects to labels list
- Label filtering on posts page shows posts with correct hierarchy

#### Search & Filtering
- Full-text search finds posts by content keywords
- Empty search shows "No results found" with helpful message
- Filter panel: label filters with OR/AND modes, author filter, date range
- Active filter chips visible when panel is closed
- Clear all removes all filters
- URL parameters update correctly with filter state

#### Dark Mode
- Three-state toggle: light -> dark -> system
- All pages render correctly in dark mode
- Code blocks use appropriate dark syntax highlighting theme
- Math formulas render with good contrast
- Tables, blockquotes, and all UI elements adapt properly

#### Admin Panel
- Settings tab: title, description, timezone with save confirmation
- Pages tab: reorder built-in pages, create custom page (About), page appears in nav
- Password tab: current/new/confirm fields with 12-char minimum
- Social tab: Connect buttons for Bluesky, Mastodon, X, Facebook

#### Edge Cases
- Empty title: Save button disabled
- Long title: Truncated to 500 characters with live counter
- Invalid label ID (spaces): 422 with pattern validation error
- Label cycle: 409 "Adding parent would create a cycle"
- Unsaved changes: beforeunload dialog blocks navigation
- Session expiry: graceful handling, redirects to login
